# Segfault Shrine UI Design Spec

## 1. 整体风格关键词

Segfault Shrine 的视觉目标是“代码错误祭坛”，不是普通工具站或 SaaS Dashboard。

关键词：

- Cyber-Occult：代码逻辑与占卜仪式融合。
- Premium Tarot：卡牌要有收藏感、仪式感和精致边框。
- High-Contrast Tech：深色终端背景配高亮强调。
- Ritualistic：通过对称布局、光效、卡牌揭晓和结果区域制造仪式感。
- Cold & Restrained：冷峻、克制、硬朗，拒绝可爱和圆润。
- Programmer Dark Humor：文案和细节带有程序员黑色幽默。

设计气质：

- 像一个用于献祭报错日志的神秘终端。
- 像一套精致但危险的 Debug 塔罗系统。
- 页面应有视觉冲击，但核心文字、卡牌和按钮必须清楚可用。

## 2. 色彩系统

### 2.1 基础颜色

| Token | 用途 | 色值 |
| --- | --- | --- |
| `--color-void` | 页面最底层背景 | `#09090B` |
| `--color-surface` | 输入框、卡片、结果面板 | `#18181B` |
| `--color-surface-raised` | 更高层级容器 | `#201F22` |
| `--color-surface-high` | hover 或激活前的层级 | `#2A2A2C` |
| `--color-stroke` | 默认边框 | `#27272A` |
| `--color-stroke-strong` | 强边框和分割 | `#4D4354` |

### 2.2 品牌与强调色

| Token | 用途 | 色值 |
| --- | --- | --- |
| `--color-oracle` | 主色、抽牌按钮、仪式触发点 | `#A855F7` |
| `--color-oracle-soft` | 主色浅层发光、卡牌高光 | `#D8B4FE` |
| `--color-data` | 技术数据、链接、光标、辅助强调 | `#22D3EE` |
| `--color-panic` | 错误、空输入、失败提示 | `#EF4444` |
| `--color-runtime` | 成功、复制完成、截图完成 | `#22C55E` |

使用规则：

- 深色中性色必须占页面大多数面积。
- 紫色用于关键动作和神秘感，不铺满整页。
- 青色用于技术数据和代码感，不与紫色抢主按钮层级。
- 不使用大面积紫蓝渐变背景。
- 状态色只在对应状态出现，避免常驻干扰。

### 2.3 文字颜色

| Token | 用途 | 色值 |
| --- | --- | --- |
| `--color-text-primary` | 标题、正文重点 | `#F4F4F5` |
| `--color-text-secondary` | 次要说明、注释 | `#A1A1AA` |
| `--color-text-muted` | 占位、弱提示 | `#71717A` |
| `--color-text-inverse` | 主按钮上的深色文字 | `#09090B` |

## 3. 字体规范

### 3.1 字体家族

- 标题与卡牌名：`EB Garamond`、`Playfair Display` 或可用衬线字体。
- UI 正文：`Inter`、`system-ui` 或可用无衬线字体。
- 代码、输入、标签、技术提示：`JetBrains Mono`、`Fira Code` 或可用等宽字体。

### 3.2 字级

| Token | 用途 | 桌面端 | 移动端 | 字重 |
| --- | --- | --- | --- | --- |
| `display` | 项目名 | 48px | 36px | 600 |
| `headline` | 区块标题、结果标题 | 32px | 28px | 500 |
| `card-title` | 卡牌名 | 24px | 22px | 600 |
| `body` | 正文说明 | 16px | 15px | 400 |
| `mono-body` | 报错输入、技术解读 | 14px | 13px | 400 |
| `label` | 标签、小按钮、状态文字 | 12px | 12px | 600 |

实现规则：

- 字体大小不随 viewport 宽度连续缩放。
- `letter-spacing` 统一为 `0`，不使用负字距。
- 等宽小标签可以通过字重、大小和颜色区分，不依赖夸张字距。
- 文本必须在按钮、卡牌和输入框内完整显示。

## 4. 间距规则

使用 4px 基础步长。

| Token | 值 | 用途 |
| --- | --- | --- |
| `space-1` | 4px | 细小间隔 |
| `space-2` | 8px | 图标与文字、标签内部 |
| `space-3` | 12px | 紧凑内边距 |
| `space-4` | 16px | 常规内边距、移动端边距 |
| `space-5` | 20px | 大控件内边距 |
| `space-6` | 24px | 模块内区块间距 |
| `space-8` | 32px | 大模块间距 |
| `space-12` | 48px | 首屏主要区块间距 |

布局规则：

- 桌面端内容最大宽度建议控制在 1120px 到 1280px。
- 桌面端主要内容左右边距至少 32px。
- 移动端左右边距 16px。
- 固定格式元素需要稳定尺寸，例如卡牌、按钮组、截图区域。

## 5. 圆角规则

整体保持硬朗，不使用大圆角或胶囊按钮。

| 元素 | 圆角 |
| --- | --- |
| 小标签、细分割容器 | 2px |
| 按钮、输入框 | 4px |
| 面板、大容器 | 6px |
| 塔罗牌 | 8px |
| 截图结果容器 | 8px |

禁止：

- 胶囊形按钮。
- 过度圆润的卡片。
- 通用移动 App 风格的大圆角。

## 6. 阴影与发光

不使用传统弥散大阴影。深度主要来自层级色、边框和克制发光。

规则：

- 默认容器只用边框，不加阴影。
- 激活卡牌可以使用紫色微光：`0 0 20px rgba(168, 85, 247, 0.35)`。
- 主按钮 hover 可以使用细微内发光或 1px 亮边。
- 错误状态使用红色边框和短暂发光，不持续大面积闪烁。
- 发光不能遮挡文字，也不能降低可读性。

## 7. 按钮样式

### 7.1 主按钮

用途：

- `抽牌` / `Draw Cards`

样式：

- 背景：`#A855F7`
- 文字：`#09090B`
- 字体：等宽或无衬线，600 weight。
- 圆角：4px。
- 边框：1px solid `#D8B4FE` 或无边框。
- 高度：桌面端 44px 到 48px，移动端至少 44px。

状态：

- Hover：增加亮边或轻微紫色发光。
- Active：亮度降低，模拟按下。
- Loading：按钮不可重复点击，显示扫描或等待状态。
- Disabled：背景变暗，文字转为 muted。

### 7.2 次要按钮

用途：

- `重新抽牌`
- `复制结果`
- `生成截图`

样式：

- 背景：透明或 `#18181B`
- 边框：1px solid `#27272A`
- 文字：`#A1A1AA`
- 圆角：4px。

状态：

- Hover：边框变为 `#A855F7` 或 `#22D3EE`。
- Success：复制或截图成功时短暂使用 `#22C55E`。
- Error：失败时短暂使用 `#EF4444`。

交互规则：

- hover 或 active 可以使用 transform，但不能造成布局跳动。
- 按钮文字不能换行挤压，也不能被图标遮挡。

## 8. 输入框和表单样式

报错输入框应像 IDE 控制台或终端输入区。

样式：

- 背景：`#09090B`
- 边框：1px solid `#27272A`
- 文字：`#F4F4F5`
- 占位：`#71717A`
- 字体：等宽字体。
- 圆角：4px。
- 最小高度：桌面端 160px，移动端 128px。

交互：

- Focus：边框变为 `#A855F7`，可带轻微紫色发光。
- Empty Error：边框变为 `#EF4444`，提示以 `[ERROR]` 开头。
- Loading：输入框视觉弱化，暂时不可编辑。

占位文案示例：

```text
// Waiting for code sacrifice...
在此献祭你的报错日志...
```

## 9. 卡牌样式

卡牌是页面最核心的视觉对象。

比例：

- 标准比例：1 : 1.6。
- 桌面端建议宽度：180px 到 240px。
- 移动端建议宽度：70vw 到 84vw，最大不超过 320px。

结构：

- 外层边框：1px solid `#27272A`。
- 内层边框：细线或几何装饰。
- 顶部：牌名。
- 中部：图案或符号区域。
- 底部：关键词和简短说明。

视觉：

- 暗色底。
- 精致几何线条。
- 可以使用电路纹、魔法阵、终端符号作为装饰。
- 紫色和青色只作为线条、高光或小面积符号。

动效：

- 3D 翻转轴心位于中心。
- 翻转过程应清晰，不要过快。
- 卡背可以使用电路板或法阵纹样。
- 动画结束后牌面必须稳定可读。

禁止：

- 卡牌太花导致文字不可读。
- 大面积玻璃拟态遮罩。
- 卡牌内使用过多不同字体。

## 10. 面板、列表与截图区域

### 10.1 解读结果面板

样式：

- 背景：`#18181B`
- 边框：1px solid `#27272A`
- 圆角：8px。
- 内边距：20px 到 24px。

内容层级：

- 占卜标题。
- 输入报错摘录。
- 三张牌组合解读。
- Debug 建议。

### 10.2 截图区域

截图应像一张可分享的 Debug 神谕海报。

必须包含：

- `Segfault Shrine`
- 报错摘录。
- 三张牌。
- 牌名和关键词。
- Debug 解读摘要。

不包含：

- 操作按钮。
- 完整长日志。
- 调试信息。
- 页面导航或浏览器外壳。

## 11. 顶部栏样式

MVP 不做复杂导航栏。

可包含：

- 左侧：`Segfault Shrine` 标识。
- 右侧：可选 GitHub / Social 链接。

样式：

- 高度紧凑。
- 背景可以是半透明深色。
- 可使用 `backdrop-filter: blur(12px)`，但不能影响性能或文字清晰度。

## 12. 状态展示

### 12.1 空状态

表现：

- 输入框显示占位符。
- 卡牌区可显示三张暗色卡背或轮廓。
- 解读区隐藏或显示短提示。

示例文案：

```text
// Waiting for code sacrifice...
```

### 12.2 加载状态

表现：

- 抽牌按钮进入扫描状态。
- 卡牌区显示卡背、虚线轮廓或仪式动画。
- 可使用紫色流光，但不要大面积闪烁。

示例文案：

```text
正在解析堆栈中的命运分支...
```

### 12.3 错误状态

表现：

- 输入框边框变红。
- 错误提示使用等宽字体。
- 提示以 `[ERROR]` 开头。

示例文案：

```text
[ERROR] 未发现可献祭的代码。
```

### 12.4 成功状态

表现：

- 复制成功或截图成功时，按钮短暂显示成功文案。
- 使用 `#22C55E` 作为短暂强调。

## 13. 移动端适配

布局：

- 单列布局。
- 标题区更紧凑。
- 输入框位于首屏上半部分，不因虚拟键盘完全遮挡抽牌按钮。
- 卡牌从横向三列改为纵向大图、横向滑动或堆叠轮播。
- 解读结果位于卡牌之后。
- 操作按钮使用全宽或底部易触达布局。

交互：

- 不依赖 hover。
- 卡牌额外交互使用 tap。
- 按钮高度至少 44px。
- 截图生成后提示用户保存或分享。

规则：

- 卡牌文字不能溢出。
- 底部操作区不能遮挡解读内容。
- 页面不能因为动画导致横向滚动。

## 14. 不允许出现的设计风格

禁止：

- 可爱、圆润、活泼插画风。
- 马卡龙色系。
- 普通 Bootstrap / 默认组件库样式。
- 过度 SaaS 化的白底卡片布局。
- 大面积紫蓝渐变背景。
- 大圆角胶囊按钮。
- 复杂弥散投影。
- 过时的重拟物 3D。
- 纯装饰性大色块遮挡内容。
- 让视觉特效压过输入、卡牌和解读的可用性。

## 15. 设计确认标准

进入开发前，设计应满足：

- 第一眼能看出这是 `Segfault Shrine`。
- 页面不像通用表单工具或 SaaS 模板。
- 三张卡牌是视觉中心。
- 输入、抽牌、结果、复制、截图的操作路径清楚。
- 深色中性背景占主导，紫色和青色只做高亮。
- 桌面端和移动端都能完成核心流程。
- 卡牌、按钮、输入框和结果面板符合本规范。
