# Segfault Shrine PRD

## 1. 项目一句话介绍

Segfault Shrine 是一个面向程序员自娱自乐的 Debug 占卜单页 Web 应用：用户输入报错信息后，系统抽出三张“玄学 Debug 塔罗牌”，并生成一段赛博神秘学风格的 Debug 解读。

## 2. 背景和目标

程序员在遇到难以复现、原因不明、看似违背常识的 Bug 时，经常会用“玄学”来调侃 Debug 过程。这个项目把这种技术圈梗做成一个可互动、可分享、视觉精致的网页体验。

第一版目标：

- 让用户可以快速输入报错信息并完成一次占卜。
- 用三张不同的 Debug 塔罗牌制造荒诞但贴近程序员语境的反馈。
- 生成一段神棍、幽默、有技术梗的 Debug 解读。
- 支持重新抽牌、复制结果、生成结果截图。
- 作为一个适合部署上线和传播的单页作品。

## 3. 目标用户

- 程序员、独立开发者、学生开发者。
- 喜欢技术梗、Debug 梗、赛博朋克和神秘学视觉的人。
- 想把报错结果生成有趣图片并分享给朋友或技术社群的人。

## 4. 核心痛点

- 遇到玄学 Bug 时，用户会有挫败感，需要一种轻松、幽默的情绪出口。
- 普通报错信息缺少戏剧性和分享感。
- 技术梗类网页如果只有文字会显得单薄，需要更强的仪式感和视觉记忆点。

## 5. MVP 功能范围

第一版必须包含：

- 单页 Web 应用，无需登录。
- 一个报错信息输入框。
- 一个“抽牌”按钮。
- 内置 36 张 Debug 塔罗牌牌库。
- 每次抽取 3 张不同的牌。
- 报错信息会影响抽牌结果；如果没有命中预设关键词，则随机抽牌。
- 抽牌后展示三张牌，三张牌分别代表三个不同的 Bug 线索。
- 展示 3D 翻牌效果。
- 根据输入内容和三张牌生成一段本地模板式 Debug 解读，不接外部 AI API。
- 支持重新抽牌。
- 支持复制结果文案。
- 支持生成结果截图。
- 支持移动端基础适配。
- 整体视觉方向为精致塔罗牌风格，具体视觉特效后续单独细化。

## 6. 非目标

第一版不做：

- 不接 OpenAI 或其他 AI API。
- 不做登录、注册、用户系统。
- 不接数据库。
- 不做支付。
- 不做后台管理。
- 不做真实代码分析或真实修复建议引擎。
- 不做多人协作、评论、点赞、社区流。
- 不做复杂历史记录管理。
- 不做完整 Figma 高保真设计系统后再开发；视觉规范会先以文档形式确定。

## 7. 用户流程

1. 用户打开 Segfault Shrine。
2. 用户在输入框中输入报错信息，例如 `Segmentation fault`、`Cannot read properties of undefined`、`Module not found`。
3. 用户点击“抽牌”。
4. 系统根据报错关键词和随机因子抽出三张不同的 Debug 塔罗牌。
5. 三张牌以 3D 翻牌形式出现。
6. 系统生成一段神棍风格 Debug 解读。
7. 用户可以复制结果文案。
8. 用户可以生成结果截图。
9. 用户可以重新抽牌。
10. 第一版部署上线后，用户可以通过公开链接访问。

## 8. 页面列表

第一版为单页应用，包含以下区域：

- 顶部标题区：展示项目名 Segfault Shrine 和一句短标语。
- 报错输入区：输入报错、日志片段或问题描述。
- 抽牌操作区：主按钮“抽牌”，抽牌后可显示“重新抽牌”。
- 卡牌展示区：展示三张不同的 Debug 塔罗牌。
- 解读结果区：展示本次占卜的 Debug 解读。
- 分享操作区：复制结果、生成结果截图。

## 9. 内容与牌库要求

第一版内置 36 张牌。每张牌至少包含：

- 牌名。
- 关键词。
- 对应 Bug 类型或 Debug 隐喻。
- 简短牌面描述。
- Debug 建议或神棍式解释素材。

抽牌规则：

- 三张牌不能重复。
- 输入内容命中特定关键词时，提高相关牌出现概率。
- 没有命中关键词时，使用随机抽取。
- 三张牌代表三个不同的 Bug 线索，而不是固定的过去、现在、未来。

## 10. 风格和气质

关键词：

- 精致塔罗牌。
- 程序员黑色幽默。
- 赛博神秘学。
- 有仪式感。
- 荒诞但像那么回事。

视觉要求：

- 第一版应优先保证塔罗牌本身精致、有收藏感。
- 视觉特效可以丰富，但不能影响核心流程可用性。
- 移动端需要能完整完成输入、抽牌、查看结果、复制和生成截图。

## 11. 验收标准

第一版完成标准：

- 用户可以输入任意报错信息。
- 点击抽牌后，可以看到三张不同的 Debug 塔罗牌。
- 报错信息能在有预设关键词时影响抽牌结果。
- 无关键词命中时仍能正常随机抽牌。
- 三张牌有 3D 翻开效果。
- 页面能生成一段神棍风格 Debug 解读。
- 用户可以重新抽牌。
- 用户可以复制结果文案。
- 用户可以生成结果截图。
- 桌面端和移动端都能完成核心流程。
- 项目可以部署上线并通过链接访问。
