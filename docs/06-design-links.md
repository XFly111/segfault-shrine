## Expected Frames / States

The prototype should cover:

- Desktop empty state.
- Desktop loading / drawing state.
- Desktop result state with three revealed cards.
- Mobile result layout.
- Empty input error state.

## Review Checklist

Before using this design as implementation reference, confirm:

- The first viewport clearly communicates `Segfault Shrine`.
- The page does not look like a generic SaaS dashboard.
- The three tarot cards are the visual center.
- The error input, draw action, interpretation, copy action, screenshot action, and redraw action are easy to find.
- The visual style matches `docs/02-design-spec.md`.
- The layout supports mobile without relying on hover.
- The screenshot result area is clear and does not include operation buttons.

## Notes

- Figma Make output is treated as a visual and interaction reference, not final production code.
- Final implementation should follow the project docs and confirmed design, not copy generated code blindly.
- If review is needed, provide exported screenshots or make sure the Figma link is publicly accessible.
